故障導引

FTB-500 光纖診斷

PM-500、FTB-7200D、FTB-5240S 都整合在這裡。先看端面，再量光，再決定 OTDR 或 OSA。

光纖結果

輸入 PM-500 讀值後按分析。

FTB-7200D OTDR 操作

  1. 先用 ViewConn Pro 看端面，髒就清潔筆清潔，清完再看一次。
  2. 接法：FTB-7200D → Launch Fiber → 被測光纖。短距離也建議加前導纖。
  3. 選波長：1310/1550；找彎折可加看 1550/1625 差異。
  4. Range 設比實際距離大；Pulse 短距離小、長距離大。
  5. 按 Start / Auto Test，保存 Trace 與 Event Table。

FTB-5240S 光譜分析

  1. 用於 DWDM/CWDM，不是 OTDR。
  2. 接到光譜測試點，確認不可過強，必要時加衰減。
  3. 看 Channel Power、波長偏移、通道是否缺失。
  4. DWDM 異常時比對 Mux/Demux、EDFA 輸入輸出與通道表。

RF / DOCSIS 診斷

RF 結果

輸入數值後開始診斷。

860 DSPi 用法

  1. 接 RF 測試點或 Tap。
  2. 選頻道/頻率，量 Power、MER、BER。
  3. DOCSIS 問題量 DS/US 與 Lock 狀態。
  4. 異常要往上游量：用戶端 → Tap → Node → 機房。

8591E 用法

  1. 接回傳測試點，必要時加 10~20 dB 衰減。
  2. 看上行頻段，觀察底噪、突波、單頻干擾。
  3. 860 告訴你嚴不嚴重；8591E 告訴你雜訊長什麼樣。
  4. 保存頻率、時間、截圖，便於抓間歇性 ingress。

工具 SOP

流程選擇

現場原則

  1. 先分範圍：單戶、整棟、整Node、整PON、全區。
  2. 先做低風險確認：告警、端面、光功率、RF數值。
  3. 不要跳步驟：髒端面會讓 PM 與 OTDR 都誤判。
  4. 每次處理都記錄「處理前 / 處理後」數據。
